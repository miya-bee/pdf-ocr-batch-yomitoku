# YomiToku Batch OCR

業務資料や自炊書籍などの**スキャンPDF**に対し、日本語OCRエンジン **YomiToku** を用いて**検索可能なテキスト層を付加する GUI アプリケーション**です。

このツールの特長は、**すでにテキスト層を持つPDFを自動判定してスキップすること**です。  
OCR済みPDFや電子生まれのPDFを再処理せず、**未処理のスキャンPDFだけを効率よく補完**できます。

---

## 主な機能

- **OCR済みPDFの自動スキップ**
  - `pypdf` を使ってPDF内の文字情報を確認し、すでにテキスト層があるPDFは処理対象から外します。

- **YomiToku による日本語OCR**
  - テキスト層を持たないPDFに対してOCRを実行し、検索可能なPDFを出力します。

- **サブフォルダ対応の一括処理**
  - 指定フォルダ配下をまとめて走査し、大量のPDFを一括処理できます。

- **GPU / CPU の切り替え対応**
  - 実行デバイスを `自動 / CPU / GPU(CUDA)` から選択できます。
  - GPUが使えない環境ではCPU運用も可能です。

- **軽量モデル対応**
  - `--lite` 相当の軽量モデルをGUIから利用できます。
  - CPU環境では軽量モデルの利用を想定しています。

- **安全な出力**
  - 標準では元ファイルを上書きしません。
  - 同名ファイルが存在する場合は、`_001` のような連番を付けて保存します。

- **CSVログ出力**
  - 処理結果、スキップ理由、出力先、処理時間などをCSVで記録できます。

---

## 想定している用途

- 過去にスキャンしたPDFのうち、OCR未処理のものだけを補完したい
- フォルダ内にOCR済みPDFと未処理PDFが混在していて、再処理を避けたい
- 日本語文書向けのOCRツールをGUIで扱いたい

---

## 動作環境

- Python 3.10 以上
- `pypdf`
- `yomitoku`
- `torch`

## インストール

まず、このリポジトリをクローンまたはダウンロードしてください。

```bash
git clone https://github.com/miya-bee/pdf-ocr-batch-yomitoku.git
cd pdf-ocr-batch-yomitoku
```

次に、依存ライブラリをインストールします。

```bash
pip install -r requirements.txt
```

### 補足

- YomiToku は **PyTorch** を必要とします。
- GPU を使う場合は、**利用している CUDA 環境に合った PyTorch** を用意してください。
- 通常モデルは GPU 向けで、CPUでも動作しますが時間がかかる場合があります。
- 軽量モデルは CPU 環境でも使いやすい構成です。
- YomiToku は**初回実行時にモデルを自動ダウンロード**するため、最初の1回はインターネット接続が必要になる場合があります。

## 使い方

1. **対象フォルダ**を選択します  
2. 必要に応じて、各オプションを設定します  
3. **実行開始**ボタンを押します

## 画面の設定項目

### 対象フォルダ
処理したいPDFが入っているフォルダを指定します。

### サブフォルダも処理する
チェックを入れると、指定フォルダ配下を再帰的に探索します。

### 既存の同名出力を上書きする
オンにすると既存出力を上書きします。  
オフの場合は、同名時に連番付きで保存します。

### 出力を元PDFと同じフォルダに保存する
オンの場合は元PDFと同じ場所に保存します。  
オフの場合は `ocr_output` フォルダに保存します。

### CSVログを出力する
処理結果の一覧をCSVで保存します。

### 実行デバイス
- **自動**: GPU(CUDA) が使えればGPU、使えなければCPUを使用
- **CPU**: CPUで実行
- **GPU(CUDA)**: GPUで実行

### 軽量モデル
軽量モデルを使ってOCRします。  
CPU環境では有効化を推奨します。

### PDF読取DPI
OCR時の読み取り解像度を指定します。  
標準値は **200** です。

---

## 出力ファイルについて

- 出力ファイル名は、元ファイル名に **`_ocr`** を付けたPDFです  
  - 例: `sample.pdf` → `sample_ocr.pdf`

- 同名ファイルがすでにある場合（上書きOFF時）は、連番で保存します  
  - 例: `sample_ocr_001.pdf`

- すでにこのツールで出力されたと見なされるファイル名は、再処理防止のためスキップされます

---

## ログについて

CSVログを有効にすると、対象フォルダ内にログファイルを出力します。  
ログには次のような情報が記録されます。

- 入力ファイル
- 出力ファイル
- 実行結果
- スキップ理由
- 使用デバイス
- 処理時間

---

## 注意事項

- このツールは**PDF専用**です
- 暗号化PDFや破損PDFは、判定またはOCR処理に失敗する場合があります
- CPUのみの環境では、通常モデルだと処理に時間がかかることがあります
- OCR精度は元PDFの画質や解像度に依存します
- 既存PDFに文字が少しでも含まれている場合、OCR済みと判定してスキップすることがあります
